# Wallet Read Product Rules

## What This Page Is

The wallet read is a cultural read of visible NFT collecting behavior.

It helps someone recognize patterns in what they collect.

## What This Page Is Not

It is not:

- a portfolio tracker
- a valuation tool
- a ranking system
- a rarity explorer
- a trading tool
- a psychological diagnosis
- a social score

## Language Rules

Use:

- suggests
- appears
- leans
- points toward
- visible signal
- collection pattern
- repeat collecting
- anchor
- archive
- room
- thread

Avoid:

- proves
- is
- best
- most valuable
- floor
- offer
- ROI
- alpha
- smart money
- whale
- elite
- ranked
- superior

## Interpretation Rules

Every interpretation should be grounded in visible wallet behavior.

Good:

> This wallet leans toward meme culture because several of its strongest collection anchors come from meme-native projects.

Bad:

> You are a meme maximalist.

## Multi-Wallet Rules

- Combined reads should feel like one collector, not several profiles stapled together.
- Wallet chips should make included sources clear.
- Source-wallet attribution should be available without dominating the page.
- Duplicate NFTs should not inflate counts.
- The first wallet is the default identity anchor unless changed later.

## Visual Rules

- Objects first.
- Avoid dashboard density.
- Avoid finance UI.
- Use collection images where available.
- Use NFT thumbnails as proof, not decoration.
- Do not render the entire wallet.
- Curate previews.

## Trust Rules

- Show proof near claims.
- Explain missing data lightly.
- Hide empty modules.
- Do not pretend classification is perfect.
- Do not overuse info icons.
